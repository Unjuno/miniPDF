# miniPDF - コンセプト文書

## プロジェクト概要

**miniPDF** は、Markdown → PDF 変換後に発生するレイアウトの「ズレ」だけを、Word感覚でサクッと直せる超軽量PDFレイアウト調整ツールです。

## 目的（解決したい問題）

Markdown → PDF 変換後に起きる以下の問題を解決します：

- Mermaid 図がデカすぎる
- 意図しない改ページ・改行
- 行が詰まらず、Word みたいに整わない

**重要な前提：**
- 本文編集は不要
- 検索可能性（テキスト検索）は維持したい
- 重いPDF編集ソフト・有料ツールは使いたくない
- CSS調整・HTML経由・複雑な設定に時間を使いたくない
- パッと開いて、パッと直して、すぐPDFにしたい

## 問題の本質

Markdown / Mermaid / PDF 変換は **「論理構造」と「最終ページレイアウト」が乖離** しています。

- HTML や Markdown 上では「どこでページが切れるか」を人間が直感的に把握できない
- PDFにしてから初めて「ズレ」が可視化される

## 設計思想

### ✅ PDFを「最終成果物」として扱う

- **PDFを直接・軽量に扱う**
- 内部構造を完全編集しない（Acrobat路線は捨てる）

### ✅ 編集対象を極限まで絞る

**やることはこれだけ：**

1. **画像（Mermaid 図）のサイズ調整**
2. **改行・改ページ位置の微調整**
3. **行間・詰まりの調整（Word的見た目）**

**やらないこと：**

- 文字内容の編集
- PDFの完全な再構造化
- 署名・フォーム・高度注釈

## 実装イメージ

### アプリの性格

- **超軽量 PDF レイアウト調整ツール**
- OSS前提
- 起動が速い（1秒以内）
- 余計な機能ゼロ

### 操作感

1. PDFを開く
2. 画面は「印刷結果そのもの（WYSIWYG）」
3. ドラッグ操作で：
   - Mermaid 図（＝PDF内画像）を拡大・縮小
   - 改ページ位置を前後にずらす
4. 行が自動で詰まる（Word的）

### 技術的割り切り

- Mermaid 図は **PDF内では画像** → **画像サイズ調整だけでOK**
- テキストは：
  - 位置は動かすが
  - 内容は触らない
  - 検索可能性は維持
- 内部編集ではなく：
  - **視覚的配置の再配置**
  - **再レンダ → 新PDF生成**

## 技術スタック

- **Tauri + WebView**：軽量・即起動・ドラッグUI向き
- レンダ：
  - PDF → 表示用レンダ
  - 編集後 → 再PDF生成
- PDFライブラリは「読む・描く」だけ（重い編集API不要）
- GitHub OSS運用：
  - Issue / PRでバグ可視化
  - README多言語化などもPRで対応可能

## なぜOSSとして価値があるか

- 同じ問題で困っている人は多い：「md→pdfにしたらズレる」
- 有料ツールは：
  - 重い
  - 高機能すぎる
  - この用途には過剰
- **「これだけ直したい」層が確実に存在**
- 小さく作れる → バグ検知しやすい → OSSと相性が良い

## 一言で言うと

> **Markdown→PDFで「最後にズレるところだけ」を、Word感覚でサクッと直せる、超軽量PDFレイアウト調整OSS**
