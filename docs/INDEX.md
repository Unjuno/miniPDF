# miniPDF - ドキュメント索引

このディレクトリは、プロジェクトのドキュメントを整理するための索引です。

## ドキュメント構成

### 1. プロジェクト概要

- **[../README.md](../README.md)** - プロジェクトの基本情報・エントリーポイント

### 2. 設計・コンセプト

- **[../CONCEPT.md](../CONCEPT.md)** - プロジェクトのコンセプト・設計思想
  - 目的と解決したい問題
  - 設計思想
  - 実装イメージ
  - 技術スタックの方向性

### 3. 仕様書

- **[../SPECIFICATION.md](../SPECIFICATION.md)** - 機能仕様・実装方法（**実装時の主要参照文書**）
  - 機能仕様（詳細な動作定義）
  - ユーザーインターフェース仕様
  - データ構造仕様
  - IPC通信仕様
  - 実装方法の決定（技術選択・アーキテクチャ）
  - モジュール設計
  - エラーハンドリング仕様
  - 非機能要件

### 4. 実装計画

- **[../IMPLEMENTATION_PLAN.md](../IMPLEMENTATION_PLAN.md)** - 実装計画書
  - 技術スタック詳細
  - アーキテクチャ設計
  - 実装フェーズ（7段階）
  - 技術的詳細設計
  - テスト方針
  - パフォーマンス目標
  - 開発マイルストーン

### 5. 技術リファレンス

- **[../TECHNICAL_REFERENCES.md](../TECHNICAL_REFERENCES.md)** - 技術リファレンス（一次情報）
  - 使用技術の公式ドキュメントURL
  - コアフレームワーク（Tauri, React, TypeScript等）
  - PDF処理ライブラリ
  - UIライブラリ・コンポーネント
  - 開発ツール

## ドキュメントの読み方

### 初めてプロジェクトを見る場合

1. **README.md** でプロジェクトの概要を把握
2. **CONCEPT.md** で設計思想と目的を理解
3. **IMPLEMENTATION_PLAN.md** で実装計画を確認

### 実装を始める場合

1. **SPECIFICATION.md** を参照（機能仕様・実装方法の決定事項）
2. **IMPLEMENTATION_PLAN.md** の「実装フェーズ」を参照
3. **TECHNICAL_REFERENCES.md** から必要な技術の公式ドキュメントを参照
4. 各フェーズの詳細を確認しながら実装

### 技術的な疑問がある場合

1. **TECHNICAL_REFERENCES.md** から該当技術の公式ドキュメントを参照
2. 公式ドキュメントで解決できない場合は、GitHub Issues等を確認

## ドキュメント更新方針

- 各文書は独立して更新可能
- 重要な変更は全ドキュメントで整合性を保つ
- 技術リファレンスは定期的に最新化
